# 测试素材生成工具 (Test Asset Generator) 设计文档

## 1. 项目概述

**项目名称:** Test Asset Generator
**项目类型:** 跨平台桌面 GUI 工具
**核心功能:** 基于 FFmpeg 生成图片、视频、音频类型的测试素材文件，支持自定义分辨率、时长、数量、格式、内容类型等参数。
**目标用户:** 组内测试人员，用于快速获取测试文件素材。

## 2. 技术方案

### 框架选择: Tauri + React

- **后端:** Tauri (Rust)，负责调用 FFmpeg、处理文件 I/O、进度管理
- **前端:** React + TypeScript，提供跨平台一致的 GUI
- **构建产物:** Windows `.exe` 安装包 + macOS `.app` 安装包，用户无需安装任何运行时
- **FFmpeg:** 内置打包，不依赖用户系统环境

### FFmpeg 打包策略

项目中存放 FFmpeg 二进制文件:
- `ffmpeg-mac` (macOS)
- `ffmpeg-win.exe` (Windows)

程序运行时根据 `std::env::consts::OS` 自动选择对应版本，调用时使用 `std::process::Command`。

## 3. 功能规格

### 3.1 素材类型

工具支持三种素材类型的独立生成:

| 类型 | 格式选项 | 默认格式 | 时长/分辨率参数 |
|------|----------|----------|------------------|
| 图片 | PNG / JPG / WEBP | PNG | 分辨率 (默认 1080x1920) |
| 音频 | MP3 / WAV / AAC | MP3 | 时长 (默认 60 秒) |
| 视频 | MP4 / MOV / WEBM | MP4 | 分辨率 (默认 1080x1920) + 时长 |

### 3.2 可配置参数

#### 图片
- 分辨率: 宽 x 高，默认 1080x1920
- 文件格式: PNG / JPG / WEBP，默认 PNG
- 内容类型: 纯色 / 渐变 / 图案(棋盘格/彩条) / 随机噪声，默认 随机噪声
- 文件数量: 1-N，默认 10
- 文件名前缀: 默认 "测试图片"

#### 音频
- 时长: 秒，默认 60
- 文件格式: MP3 / WAV / AAC，默认 MP3
- 采样率: 44100 / 48000 Hz，默认 44100
- 声道: 单声道 / 立体声，默认单声道
- 文件数量: 1-N，默认 10
- 文件名前缀: 默认 "测试音频"

#### 视频
- 分辨率: 宽 x 高，默认 1080x1920
- 帧率: 30 / 60，默认 30
- 编码格式: H.264 / H.265，默认 H.265
- 时长: 秒，默认 60
- 内容类型: 纯色 / 渐变 / 图案(棋盘格/彩条) / 随机噪声，默认 随机噪声
- 文件数量: 1-N，默认 10
- 文件名前缀: 默认 "测试视频"

#### 全局参数
- 文件数量、分辨率、时长在图片/音频/视频标签页各自独立设置
- 用户修改默认配置后，自动沿用该配置（持久化到本地配置）

### 3.3 文件命名规则

格式: `{前缀}_{随机字符串}_{序号}.{扩展名}`
示例: `测试图片_a02j6f1_001.png`

- 随机字符串: 6-8 位字母数字组合，使用 Rust `rand` crate 生成
- 序号: 001, 002, ... N，使用零填充对齐

### 3.4 内容生成方式

基于 FFmpeg filter 生成:

| 内容类型 | FFmpeg 实现方式 |
|----------|----------------|
| 纯色 | `color=c={color}:s={w}x{h}:d={duration}` |
| 渐变 | `gradients` 或 `mandelbrot` |
| 图案-棋盘格 | `cellauto` + `scale` |
| 图案-彩条 | `testsrc2` |
| 随机噪声 | `anoisesrc` (音频) / `cellauto` + 随机种子 (视频) |

每生成一个文件，通过改变 seed/颜色参数确保 MD5 不同。

### 3.5 路径管理

- **首次运行:** 弹出路径选择器，让用户选择默认保存路径
- **后续运行:** 自动使用上次保存的路径（持久化到本地配置）
- **每次任务:** 在保存路径下新建子文件夹，命名格式: `YYYY-MM-DD-HH-mm-ss`（如 `2026-04-15-13-30-45`），避免文件混杂

### 3.6 进度与状态

#### 生成前
- 显示预计文件体积（根据分辨率、时长、格式、码率估算）
- 显示文件数量

#### 生成中
- 进度条（已完成文件数 / 总文件数）
- 预计剩余时间
- 当前正在生成的文件名
- 取消按钮

#### 生成后
- 汇总信息: 成功 N 个，失败 N 个
- 失败时显示具体错误信息（FFmpeg stderr 输出）
- 已生成的文件全部保留（即使中途取消或出错）

### 3.7 错误处理

- 单个文件生成失败 → 跳过，继续生成剩余文件
- 严重错误（如磁盘满） → 中断任务，保留已生成文件，显示错误信息
- 不检查磁盘剩余空间

## 4. 数据流

```
用户配置 (GUI)
    │
    ▼
Tauri Command (Rust)
    │
    ├── 估算文件体积
    │
    ├── 创建时间戳子目录
    │
    └── 循环生成文件:
            │
            ├── 生成随机种子/颜色参数
            ├── 构建 FFmpeg 命令
            ├── 执行 Command (实时 stderr)
            ├── 检查退出码
            │       ├── 成功 → 继续
            │       └── 失败 → 记录错误，跳过
            │
            └── 更新进度 (emit 事件到前端)
```

## 8. MD5 唯一性保证

每个生成的文件通过以下方式确保 MD5 不同:

- **纯色:** 每次随机生成一个 RGB 颜色值（色相 H 在 0-360° 随机）
- **渐变:** 随机改变起始/终止颜色 + 角度
- **图案:** 随机种子值
- **随机噪声:** 随机种子 + 随机振幅

配合文件名中的随机字符串，MD5 冲突概率可忽略不计。

- [ ] 视频支持 MP4/MOV/WEBM，分辨率/帧率/编码可选

- [ ] 首次选择保存路径，后续默认使用
- [ ] 每次任务新建时间戳子文件夹
- [ ] 用户修改的配置自动保存并沿用
- [ ] FFmpeg 无需用户额外安装
